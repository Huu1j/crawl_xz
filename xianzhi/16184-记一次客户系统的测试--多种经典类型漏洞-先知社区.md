### SQL注入

登入系统后，通过瞎注自动插入引号判断，发现注入。系统内有很多注入点，我这边就挑一个给大家展开说一下子。

![](images/20241214222900-c2dff1ca-ba27-1.png)  
接下来验证注入，这里我就不贴SQLMap的图了，最开始梭哈过，跑着跑着卡住不动了，不过这也不是第一次了，习惯了……直接手工来吧。通过响应包可见，一个单引号与两个单引号时，响应返回内容不一致，说明单引号可以对其产生影响：

![](images/20241214232547-b176d7ac-ba2f-1.png)

![](images/20241214232652-d814c5a4-ba2f-1.png)  
但是，现在并不能确定单引号就是闭合符合，所以我们构造语句继续确认，语句如下：

```
1'||'1'='1
1'||'1'='2

```

![](images/20241214223704-e36f4b38-ba28-1.png)  
啊这，正常来说如果是单引号闭合的话，上面两条语句返回的内容应该是不一致的，但是上面的语句并未生效。这说明，闭合符号可能不是单引号。那么，有可能存在括号，继续构造语句：

```
1')||(1='1
1')||(1='2

```

![](images/20241214232259-4d64712a-ba2f-1.png)

![](images/20241214232408-764f47a4-ba2f-1.png)

OK，果然带括号，构造语句确认到数据库名长度为7：

![](images/20241214231956-e01596b2-ba2e-1.png)  
接下来我们继续构造语句，来获取库名：

```
1')||(substr('abc',1,1)='a')and(1='1
1')||(substr(database(),1,1)='X')and(1='1

```

![](images/20241214225604-8a77d4c0-ba2b-1.png)
