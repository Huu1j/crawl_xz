有很多小伙伴对调试so文件都有疑惑，网上很多都是不是很全或者很复杂，这里详细说明到底如何能够调试起so文件。

这里我们介绍如何调试so 文件：

这是一个大坑，每次都忘！  
先启动android\_server

我的机器是：

```
adb shell

cd data/local/tmp

./android_server64
```

![image.png](images/img_17813_000.png)

这个窗口不能关闭，一直开着：重新打开一个窗口：

进行端口转发：

```
adb forward tcp:23946 tcp:23946
```

使得PC端的IDA连上这个端口

​

现在我们需要将apk安装到模拟器/真机中：执行命令：

```
adb devices

adb install   xx.apk
```

安装好以后，通过aapt命令获取包名：

执行下面的命令：

```
aapt dump badging xx.apk
```

![image.png](images/img_17813_001.png)

这里就是我们要的包名：com.moible.midand.MainActivity

然后开启调试模式：

```
adb shell am start -D -n com.moible.midand/.MainActivity
```

*注意一定是包名/类名*

​

这里真机/模拟器会出现Debugger wait

不用动。

然后利用IDA加载so文件：配置Debugger:

![image.png](images/img_17813_002.png)

这里一定要选择附加进程（attach to process）

![image.png](images/img_17813_003.png)

我们寻找之前的包名：即：adb shell am start -D -n com.moible.midand/.MainActivity

中的com.moible.midand

![image.png](images/img_17813_004.png)

到这个地方以后：

这里会卡住：

使用ddms获取端口号:

![image.png](images/img_17813_005.png)

![image.png](images/img_17813_006.png)

有小虫子的就是正在调试：  
这里显示的是8700

![image.png](images/img_17813_007.png)

运行命令：

```
jdb -connect com.sun.jdi.SocketAttach:hostname=127.0.0.1,port=8700
```

达到这样的小国就可以了，记得一开始要下断点哦！

![image.png](images/img_17813_008.png)

后面的和正常调试是一样的。
