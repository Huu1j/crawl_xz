#### mips相关知识

在此之前需要对`mips`架构下的汇编有一定的理解，这里说明一下`mips`架构下一些和`amd`架构不同的地方，`mips`架构在函数调用的时候如果是叶子函数（即这个函数里面没有调用其他的函数）它的返回地址会保存至`$ra`里面，但是如果它调用了多个地址那么会把它的返回地址放入栈中，因为`$ra`肯定要存储子函数的返回地址等等，当然在此之前还会把参数`s0-s7`依次放入栈中，所有在溢出的时候可以合理的控制参数来实现`ROP`,类似下面这种会之间把栈上的偏移赋值给寄存器

![image-20250116104230381.png](images/img_17924_022.png)

但是对于32位`mips`架构来说前面的参数在`a0-a3`寄存器中，多余的参数会通过`lw`放在栈上,因此还需要找一些gadget来实现参数的转移，比如`move`指令等等

类似于下面这种

![image-20250116104744776.png](images/img_17924_023.png)

还有一个特性就是**流水线效应**，举个例子，就拿上面这个图来说\*\*，jalr**跳转命令的下一条命令为**分支延迟槽\*\*，也就是说像这种跳转指令在跳转的时候会出现分支延迟效应，在跳转的时候下一个指令也已经被执行了，在例子中就是在跳转到`$t9`之前，`$t2`的值会给`$a2`,这样的话找`gadget`的时候又会多一种选择，了解完这个就可以开始测试具体偏移了

#### 系统模式下模拟

要想在系统模式下进行模拟，**首先需要下载对应的镜像以及内核**

[Index of /~aurel32/qemu/mipsel](https://people.debian.org/~aurel32/qemu/mipsel/)

到上面这个网站下下 `debian_squeeze_mipsel_standard.qcow2`镜像以及 `vmlinux-3.2.0-4-4kc-malta` 内核

在开始模拟之前需要开启物理机的ip转发功能，以及创建网桥来连接到`qemu`虚拟机

安装网络配置工具：`apt-get install bridge-utils uml-utilities`  
开启：`sysctl -w net.ipv4.ip_forward=1`

配置一下`/etc/network/interfaces`文件

```
auto lo
iface lo inet loopback

auto ens33
iface ens33 inet manual
    up ifconfig ens33 0.0.0.0 up

auto br0
iface br0 inet dhcp
    bridge_ports ens33
    bridge_maxwait 0
```

这里根据情况来定，如果宿主机上面是ens33那就不用修改如果是eth0那就把ens33换成eth0即可

下面是启动脚本，`start.sh chmo+x` 加上执行权限

```
#!/bin/bash
sudo qemu-system-mipsel \
    -M malta -kernel vmlinux-3.2.0-4-4kc-malta \
    -hda debian_squeeze_mipsel_standard.qcow2 \
    -append "root=/dev/sda1 console=tty0" \
    -net nic,macaddr=00:16:3e:00:00:01 \
    -net tap
```

启动起来之后通过`scp` 命令把文件系统copy过去，可能会出现报错，因为在老版本中需要指定一下算法`-o HostkeyAlgorithms=ssh-rsa`

之后就正常`ssh`即可，如果是第一次启动需要`ip a`看一下qeum虚拟机对应的网卡，应该是eth1 然后配置一下`/etc/network/interfaces`文件修改成对应的网卡名称即可
