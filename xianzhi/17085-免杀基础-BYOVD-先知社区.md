Bring Your Own Vulnerable Driver的简称BYOVD 是一种滥用带签名驱动 执行0环权限操作 达到恶意攻击的技术

实际上就是白驱利用

想要利用该技术 涉及到驱动的一些基础知识

# 驱动通信

常规驱动的通信方法是基于设备对象的(\_DEVICE\_OBJECT) 将信息封装成 \_IRP 进行传输

其中驱动对象存在一个叫MajorFunction(派遣函数)的成员 这是一个数组 存储着通信的回调函数 当用户模式调用了特定函数后会触发对应的回调函数

![](images/20250307140635-533b263b-fb1a-1.png)

IRP\_MJ\_CREATE 处理打开设备时的操作

IRP\_MJ\_CLOSE 处理关闭设备时的操作

IRP\_MJ\_DEVICE\_CONTROL 处理IOCTL

```
NTSTATUS
 createRoutine(
     _In_ struct _DEVICE_OBJECT* DeviceObject,
     _Inout_ struct _IRP* Irp
 ) {
 
     DbgPrintEx(77, 0, "create!
");
     return STATUS_SUCCESS;
 }
 
 ....
 DriverObject->MajorFunction[IRP_MJ_CREATE] = createRoutine;
```

这样就注册成功了
