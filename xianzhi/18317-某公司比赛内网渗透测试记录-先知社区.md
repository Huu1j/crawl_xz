# 第三层

现在就是正常的渗透，对网卡进行收集，以及对其进行扫描

![image.png](images/20250626153149-9f56e10c-525f-1.png)

发现第二张网卡的信息

```
10.10.10.60/24
```

老样子，对其进行扫描

![image.png](images/20250626153149-9f75a81c-525f-1.png)

发现其10.10.10.70上面的一个端口存在activemq的默认密码，这种其实已经可想到activemq的cve漏洞

先对其控制的172.16.200.60上弄一个隧道代理，然后用bp去访问这个服务

![image.png](images/20250626153150-9f92d450-525f-1.png)

![image.png](images/20250626153150-9fa88e1c-525f-1.png)

然后配置bp，去访问

![image.png](images/20250626153150-9fc3017a-525f-1.png)

默认密码

```
admin/admin
```

![image.png](images/20250626153150-9fefa3ec-525f-1.png)

然后搜索此服务的cve漏洞

![image.png](images/20250626153151-a02527a6-525f-1.png)

直接利用即可

先把kali的代理配置好

![image.png](images/20250626153151-a0481630-525f-1.png)

然后在已经控制的靶机的服务器上搭建一个http服务，利用python

![image.png](images/20250626153151-a0565dc6-525f-1.png)

然后再利用poc

```

```

![image.png](images/20250626153151-a09a0526-525f-1.png)

记得在10.10.10.60上监听4444端口

![image.png](images/20250626153152-a0c107b6-525f-1.png)

然后执行poc

![image.png](images/20250626153152-a0d3a2f4-525f-1.png)

成功反弹，在根目录下获得flag

```
flag7{5b458b1b-6668-42f7-9307-657941f3a0b0}
```
