# 二、分析各种技术方法

**1.依赖于第三方资源**

<https://github.com/wwl012345/CDNCheck>

爬取全球ping，感觉不够优雅

<https://github.com/YouChenJun/CheckCdn>  
依托于五大云厂商，需要有余额，介绍说可以退费。笔者嫌一个个需要配置，并未实验检测其具体效果

<https://github.com/timwhitez/Frog-checkCDN>

依赖于维护的cdn资产，还有<https://github.com/zu1k/nali>和<https://github.com/projectdiscovery/cdncheck>

付费查询<https://cdn.chinaz.com/>

**2.DNS信息查询**

<https://github.com/alwaystest18/cdnChecker>和<https://github.com/ccc-f/FCDN>的技术方法对笔者完成本项目帮助很大，特此感谢。

通过查询A记录获取解析到的IP

查询CNAME记录，可以理解为域名的别名，使用了waf或cdn，大概率会存在服务商提供的域名，比如zjuss-cn.cname.saaswaf.com，故可以作为判断的依据

这样仅通过DNS请求信息即可，不依赖第三方服务，收集到的也是最新信息

# 三、**设计判断cdn逻辑**
