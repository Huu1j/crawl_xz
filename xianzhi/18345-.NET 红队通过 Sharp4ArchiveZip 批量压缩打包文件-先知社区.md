## 0x01 SharpZipLib 基本介绍

SharpZipLib 全称为 ICSharpCode.SharpZipLib，是一个开源的专为 .NET 平台设计的压缩与解压缩库。最早由开源 IDE 项目 SharpDevelop 团队维护，后来逐步发展为 .NET 生态中极为流行的压缩处理组件之一。

​

![](images/20250627152137-5d09e286-5327-1.png!post)

该组件提供了对多种主流压缩格式的支持，包括但不限于：ZIP、GZIP、TAR，既支持压缩单个文件、目录递归打包、流压缩处理，也支持密码保护、压缩等级设置等高级功能，因此被广泛应用于日志归档、自动打包、跨平台数据传输等场景中。

使用时推荐通过 NuGet 进行安装，

访问地址：<https://www.nuget.org/packages/SharpZipLib>

具体的安装命令如下所示。

​

```
dotnet add package SharpZipLib --version 1.4.2
```

​

运行命令后将组件添加至当前的 .NET 应用项目。该组件最常用的是 CSharpCode.SharpZipLib.Zip 命名空间，主要用于 ZIP 格式相关操作，如 ZipFile, ZipOutputStream, ZipEntry ，可以方便地将整个文件夹打包为 .zip 文件。而 ICSharpCode.SharpZipLib.GZip 空间下的类用于 GZIP 格式处理，常见于日志压缩。

## 0x02 ZipOutPutStream

ZipOutputStream 是 ICSharpCode.SharpZipLib.Zip 命名空间中提供的一个核心类，它用于以流的方式创建 ZIP 压缩文件。在实际使用中，ZipOutputStream 通常用于将多个文件或目录逐一写入一个 .zip 文件中，并支持自定义压缩等级、密码保护等功能。一般使用方法如下代码所示。

​

```
using (ZipOutputStream zos = new ZipOutputStream(File.Create("output.zip")))
{
    zos.Password = "123456";  // ZIP 文件使用弱加密（ZIPCrypto）
    zos.SetLevel(5); // 压缩等级 0（无压缩）到 9（最高压缩）
    string filePath = @"C:\test.txt";
    byte[] buffer = File.ReadAllBytes(filePath);
    // 创建一个压缩包条目
    ZipEntry entry = new ZipEntry("test.txt");
    zos.PutNextEntry(entry); // 准备写入
    zos.Write(buffer, 0, buffer.Length); // 写入内容
    zos.Finish(); 
}
```

相比直接使用 ZipFile.Create 或 System.IO.Compression.ZipArchive，ZipOutputStream 提供了更底层、流式、可控的压缩能力 。

## 0x03 ZipEntry

ZipEntry 是 ICSharpCode.SharpZipLib.Zip 命名空间下的一个重要类，表示 ZIP 文件中的一个条目，也就是压缩包中的一个文件或文件夹的元信息。在 .zip 文件结构中，每一个被压缩进去的文件或目录，都会有一个对应的条目。这些条目的描述信息包括：文件名、是否是目录、修改时间、原始大小、压缩后大小，而这些元信息，就是通过 ZipEntry 这个类来表示和操作的。

简单来说： 一个 ZIP 文件由多个 ZipEntry 构成，每个 ZipEntry 代表其中的一个压缩对象，可以是文件，也可以是空目录，该类一般使用方法如下代码所示。

​

```
using (ZipOutputStream zos = new ZipOutputStream(File.Create("output.zip")))
{
    zos.SetLevel(9); 
    string filePath = @"C:\temp\hello.txt";
    byte[] fileBytes = File.ReadAllBytes(filePath);
    // 创建 ZipEntry，并设置文件在压缩包中的路径
    ZipEntry entry = new ZipEntry("docs/hello.txt")
    {
        DateTime = DateTime.Now,
        Size = fileBytes.Length
    };
    zos.PutNextEntry(entry);        // 添加条目头
    zos.Write(fileBytes, 0, fileBytes.Length); // 写入内容
    zos.CloseEntry();               // 关闭条目
}
```

上述代码中需要注意 ZipEntry 的 Name 属性 必须使用相对路径，不要使用绝对路径；添加目录时，路径 应以 / 结尾，例如 "images/"，否则不会被识别为目录。

## 0x04 编码实现

我们以 Sharp4ArchiveZip.exe 为例，该工具是基于 ICSharpCode.SharpZipLib 实现的目录压缩工具。具体用法如下所示。

​

```
Sharp4ArchiveZip.exe "D:\VisualStudioItem\Sharp4ZipPlus\Sharp4AOT" "D:\VisualStudioItem\Sharp4ZipPlus\123.zip" "obj" ".user"
```

第一个参数：原始目录路径；第二个参数：压缩后 zip 文件输出路径；第三个参数：排除的子目录名称，多个以英文逗号分隔；第四个参数：排除的文件扩展名。

​

该命令将把 Sharp4AOT 目录中的所有文件，排除 obj 子目录 和 .user 文件，压缩为一个 123.zip 文件。下面结合源代码我们一起解读下，首先，能够递归地压缩整个目录及其所有子目录，无需手动指定每个路径。具体代码如下所示。

​

```
private static void addZipEntry(string PathStr, ZipOutputStream zos, string BaseDirName , string excludedDirectories, string excludedExtensions)
{
    DirectoryInfo dir = new DirectoryInfo(PathStr);
    string[] excludedExtensionArray = excludedExtensions.Split(',');
    string[] excludedDirectoryArray = excludedDirectories.Split(',');
    foreach (FileSystemInfo item in dir.GetFileSystemInfos())
    {
        if ((item.Attributes & FileAttributes.Directory) == FileAttributes.Directory)//如果是文件夹继续递归
        {
            // Check if the directory should be excluded
            if (Array.Exists(excludedDirectoryArray, dirName => string.Equals(item.Name, dirName.Trim(), StringComparison.OrdinalIgnoreCase)))
            {
                continue;
            }
            
            addZipEntry(item.FullName, zos, BaseDirName, excludedDirectories, excludedExtensions);
        }
        else
        {
            FileInfo f_item = (FileInfo)item;
            if (Array.Exists(excludedExtensionArray, ext => string.Equals(f_item.Extension, ext.Trim(), StringComparison.OrdinalIgnoreCase)))
            {
                continue;
            }
            using (FileStream fs = f_item.OpenRead())
            {
                byte[] buffer = new byte[(int)fs.Length];
                fs.Position = 0;
                fs.Read(buffer, 0, buffer.Length);
                fs.Close();
                ZipEntry z_entry = new ZipEntry(item.FullName.Replace(BaseDirName, ""));
                zos.PutNextEntry(z_entry);
                zos.Write(buffer, 0, buffer.Length);
                Console.WriteLine("[+]: " + item.FullName);
            }
        }
    }
}
```

代码中，用户可以自定义排除某些目录，例如 obj 或 .git 文件夹，避免不必要的冗余内容被打包；同时，它也支持排除特定的文件扩展名，例如 .user 或 .pdb 文件，进一步精简压缩包的体积。排除逻辑如下代码所示：

​

```
if (Array.Exists(excludedDirectoryArray, dirName => item.Name.Equals(dirName.Trim())))
    continue;
if (Array.Exists(excludedExtensionArray, ext => f_item.Extension.Equals(ext.Trim())))
    continue;
```

符合要求的文件则读入内存，写入 ZipEntry，通过这种方式，红队人员可以实现灵活的自定义打包策略。

​

```
using (FileStream fs = f_item.OpenRead())
{
    byte[] buffer = new byte[(int)fs.Length];
    fs.Read(buffer, 0, buffer.Length);
    ZipEntry z_entry = new ZipEntry(item.FullName.Replace(BaseDirName, ""));
    zos.PutNextEntry(z_entry);
    zos.Write(buffer, 0, buffer.Length);
} 
```
